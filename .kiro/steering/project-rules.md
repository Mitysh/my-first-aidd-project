# Правила проекта

## Структура проекта

- Исходный код — в корне или в `src/` (по мере роста проекта).
- Спецификации и дизайн-документы — в `docs/`.
- `PROJECT_STATUS.md` — в корне, отражает актуальное состояние проекта.
- `README.md` — в корне, описание проекта для GitHub.

## Работа с задачами

- Каждая задача оформляется как issue на GitHub.
- Для каждой задачи создаётся отдельная ветка от актуальной main.
- Формат ветки: `тип/номер-краткое-описание` (например `feat/4-user-profile`).
- В ветке создаётся спецификация в `docs/` и код.
- По готовности — PR с описанием, squash and merge в main.

## Стек

- HTML, CSS, JavaScript (vanilla, без фреймворков).
- Всё в одном файле допустимо для простых фич; при росте — разделять на файлы.

## Качество

- Код должен работать сразу при открытии в браузере, без сборки.
- Дизайн — минималистичный и приятный.
- Комментарии в коде — только где действительно нужны.
